## Prompt 撰寫技巧

### 基本原則

> [!tip] 清楚 > 聰明
> 不需要用花俏的技巧。清楚說明**你是誰、目標是什麼、有什麼限制**，效果遠比複雜的 prompt 好。

**好 prompt 的結構：**
```
角色設定（可選）：你是一個有 10 年經驗的 Python 工程師
任務描述：幫我 review 這段 API 程式碼
背景資訊：這個 API 會被公開使用，需要考慮安全性
限制條件：只指出重要問題，不要風格建議
輸出格式：條列式，按嚴重程度排序
```

### Chain of Thought（思維鏈）

要求 AI 一步一步思考，適合複雜推理問題。

```
❌ 不好：這個演算法的時間複雜度是多少？

✅ 好：請一步一步分析這個演算法的時間複雜度，
       說明每個迴圈的執行次數，最後給出結論。
```

### Few-shot（範例引導）

給 AI 看你要的格式範例。

```
請將以下功能需求轉換為 GitHub Issue 格式。

範例：
輸入：使用者需要能夠重設密碼
輸出：
  標題：[Feature] 密碼重設功能
  描述：使用者應可透過 email 收到重設連結...
  AC：
  - [ ] 輸入 email 後收到重設信
  - [ ] 連結 24 小時內有效

現在請處理：使用者需要能夠匯出資料為 CSV
```

## Claude Code 工作流技巧

### 善用 CLAUDE.md

在每個專案建立 `CLAUDE.md`，省去每次重複說明。

```markdown
# 專案名稱

## 技術棧
- 語言/框架版本

## 開發規範
- 命名規則
- 必須的 type hints / 測試要求

## 常用指令
- 啟動：xxx
- 測試：xxx
- 部署：xxx

## 注意事項
- 不要做 XXX
- 重要的架構決策說明
```

### Subagent 使用時機

```
✅ 適合用 subagent：
- "幫我研究這個 library 的文件"（會產生大量讀取結果）
- "掃描整個 repo 找所有 TODO"（會讀很多檔案）
- 可平行的獨立任務

❌ 不適合用 subagent：
- 需要在主對話中持續使用結果的任務
- 簡單的單一步驟任務
```

### Context 管理

> [!warning] 對話太長時的處理
> 當對話很長，AI 對早期資訊的注意力會下降。
> 定期開新對話，或使用 `/clear` 清除歷史。

**保持 context 乾淨的習慣：**
1. 複雜探索任務交給 Subagent
2. 一個對話只做一件事
3. 用 `CLAUDE.md` 代替在對話中重複說明規則

## 常見 AI 使用場景

### 程式碼開發

| 任務 | 建議做法 |
|---|---|
| 寫新功能 | 先說明架構，給相關現有程式碼，再請 AI 實作 |
| Debug | 貼錯誤訊息 + 相關程式碼 + 你已嘗試過的方法 |
| Code Review | 用 `/review` 或貼程式碼請 AI 指出問題 |
| 寫測試 | 先給 AI 看被測試的程式碼，說明預期行為 |
| 重構 | 說明重構目標，確認 AI 理解後再執行 |

### 文件與寫作

| 任務 | 建議做法 |
|---|---|
| 技術文件 | 給程式碼 + 目標讀者，AI 自動產生說明 |
| Commit message | 給 diff，AI 分析變更意圖 |
| PR 描述 | 給 commit 歷史 + 改動說明 |

### 學習與理解

| 任務 | 建議做法 |
|---|---|
| 理解陌生程式碼 | "假設我是這個領域的新手，解釋這段程式碼" |
| 學新技術 | "用類比的方式解釋，我熟悉 Python" |
| 評估方案 | "列出 A 和 B 方案的優缺點，重點在維護性" |

## 避免常見錯誤

> [!warning] 不要盲目接受 AI 的回答
> - 程式碼：一定要測試
> - 事實：重要的請查證來源
> - 安全相關：務必請資安專業人員審查

> [!warning] 不要把敏感資訊給 AI
> - API keys、密碼、個人資料
> - 公司機密、未公開的財務資訊
> - 客戶的個人識別資訊（PII）

> [!warning] 不要讓 AI 做不可逆的操作
> - 刪除資料前先確認
> - 部署到正式環境前先在測試環境驗證
> - git force push 等危險操作要謹慎

## 提升效率的設定

### 推薦的 Hooks 設定

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "command": "# 寫完 Python 檔後自動格式化",
        "condition": "$CLAUDE_TOOL_INPUT_PATH ends with .py",
        "command": "black $CLAUDE_TOOL_INPUT_PATH"
      }
    ],
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "# 阻止危險指令",
        "condition": "$CLAUDE_TOOL_INPUT_COMMAND contains 'rm -rf'",
        "exit_code": 1
      }
    ]
  }
}
```

### 推薦的 MCP 組合

**一般開發者：**
- `github` — PR / Issue 管理
- `context7` — 自動抓最新文件
- `filesystem` — 本地檔案操作

**全端工程師加：**
- `playwright` — 前端測試自動化

**有專案管理需求加：**
- `linear` 或 `asana`
